📋 Study Dashboard - Project & Session Guide

Understanding CLAUDE.md hierarchy and project context loading

1 How Claude Knows Which CLAUDE.md to Load

Automatic Detection

Claude automatically detects your current working directory and loads all relevant CLAUDE.md files up the directory tree.

ℹ️ Key Point: The working directory determines which CLAUDE.md files are active. When you navigate to a project, the right files load automatically.

2 CLAUDE.md Hierarchy (For study-dashboard)

When working in: C:\development\paul\study-dashboard\

📚 CLAUDE.md Files Loaded (Top to Bottom):
1️⃣ C:\Users\User\.claude\CLAUDE.md
↳ Global user instructions (applies to ALL projects)
↓
2️⃣ C:\development\CLAUDE.md
↳ Workspace root instructions (applies to all projects under C:\development)
↓
3️⃣ C:\development\paul\study-dashboard\CLAUDE.md
↳ Project-specific instructions (applies ONLY to study-dashboard)
✓ All Three Are Active: If there's a conflict between them, the most specific (deepest) one wins.

3 What Each Level Contains

Global CLAUDE.md (User Level)

Location: C:\Users\User\.claude\CLAUDE.md

  • Your role and background (DBA, Python scripting)
  • Workspace location preferences (C:\development)
  • General preferences (short responses, no filler)
  • Default tools (Python for scripts, Oracle/PostgreSQL)

Workspace Root CLAUDE.md

Location: C:\development\CLAUDE.md

  • Overview of all projects under C:\development
  • Directory structure (paul/, khailer/)
  • Multi-project guidelines and conventions
  • Shared workspace preferences

Project-Specific CLAUDE.md

Location: C:\development\paul\study-dashboard\CLAUDE.md

  • Study-dashboard specific details
  • Key files and their purposes
  • Commands to use (npm run validate, npx serve)
  • Chapter HTML file locations and patterns
  • Firebase and service worker setup
  • How to add chapters and subjects

4 What Happens When You Switch Projects

Scenario: Working on Different Projects

📍 Currently in: study-dashboard

C:\development\paul\study-dashboard\
CLAUDE.md files loaded:
✓ Global
✓ Workspace root
✓ study-dashboard (project-specific)
↓ (You navigate to a different project)

📍 Switch to: repo_python

C:\development\paul\github\repo_python\
CLAUDE.md files loaded:
✓ Global (same)
✓ Workspace root (same)
⚠️ repo_python (project-specific) ← Different!
✓ What changes: Global and workspace root stay the same. Only the project-specific CLAUDE.md changes based on which project directory you're in.

5 Comparison: What's Automatic vs. Manual

✅ Automatically Loaded

  • All CLAUDE.md files (detected by working directory)
  • System context (these instructions)
  • Current session history

No action needed. Claude loads these at session start.

📖 You Should Check

  • Memory files (project state, preferences)
  • Git status (recent changes)
  • Project guides (workflow, setup)

Brief review. Reference these to stay oriented.

6 System Context Display

At the top of every session, you'll see:

Contents of C:\Users\User\.claude\CLAUDE.md Contents of C:\development\CLAUDE.md Contents of C:\development\paul\study-dashboard\CLAUDE.md

This shows: All three CLAUDE.md files are loaded and active. They're displayed in order (global → workspace → project-specific).

7 Memory Files vs. CLAUDE.md

Key Difference

CLAUDE.md

Content: Project instructions, commands, patterns, setup

Updates: You edit these files, they're version-controlled in Git

Loading: Automatic (based on working directory)

Scope: Global, workspace, or project-specific

Memory Files

Content: Project state, feedback, technical solutions, automation status

Updates: Claude updates these based on conversations

Loading: Persistent across sessions (you should check them)

Scope: Project-specific only

8 Complete Picture: Session Startup

When You Start a New Claude Code Session

  1. Working directory detected: Claude sees you're in C:\development\paul\study-dashboard\
  2. CLAUDE.md files loaded: Global + Workspace + Project-specific (all active)
  3. System context shown: You see all three CLAUDE.md files at top of chat
  4. Memory system ready: Your persistent memory files are accessible
  5. You're ready to work: Ask Claude what you need to do

💡 Quick Reference

Remember:

✓ CLAUDE.md is automatic (working directory detection)
✓ All three levels are loaded (global, workspace, project)
✓ Memory files persist (check them for project state)
✓ You don't need to ask Claude to read CLAUDE.md
✓ Project-specific CLAUDE.md overrides others on conflict
