🔧 Study Dashboard - Environment Setup

Complete guide to set up development environment

1 System Requirements

Required Software

SoftwareVersionPurpose
Node.js≥ 18.0.0JavaScript runtime
npm≥ 9.0.0Package manager
GitLatestVersion control

Check versions:

node --version && npm --version && git --version

2 Clone Repository

Get the code:

git clone https://github.com/pp-nextgen-dba/study-dashboard.git
cd study-dashboard

3 Install Dependencies

Install npm packages:

npm install

Installs: docx (for document support) and other tools

4 Project Structure

Key directories:

  • js/subject.js - Master data (14 subjects, chapters)
  • js/firebase.js - Firebase config (hardcoded, pre-configured)
  • subjects/ - Subject pages (biology.html, chemistry.html, etc.)
  • chemistry/, biology/, english/ - Root-level chapter HTML files
  • sw.js - Service worker for offline caching
  • CLAUDE.md - Project documentation
  • html-guide/ - Guide pages (this file)

5 Available Commands

npm run validate

Validates all subject data, registries, and HTML pages for consistency.

npm run validate

npm run check-codes

Shows next available 3-digit codes for English (en_), Malay (my_), and Chinese (zh_) chapters.

npm run check-codes

npx serve

Run local development server on port 3000.

npx serve .

Then open: http://localhost:3000

6 Firebase & Authentication

Pre-configured:

  • Firebase config is hardcoded in js/firebase.js
  • Anonymous authentication enabled (no login required)
  • Firestore database stores progress (form, status, confidence)
  • Auto-syncs across browser sessions
💡 Important: When adding chapters with resourceUrl, the Firestore sync function (addMissingChineseLabels) keeps data consistent.

7 Service Worker & Caching

What it does:

  • Caches static assets (HTML, CSS, JS) for offline access
  • Firestore requests bypass cache (always fresh)
  • Enables PWA (installable on desktop/mobile)

When adding chapters:

Update STATIC_ASSETS in sw.js with new chapter paths:

const STATIC_ASSETS = [ '/index.html', '/subjects/biology.html', '/biology/biology_chapter8.html', // ← Add new chapters here ];

8 Git Workflow

Your workflow involves:

  • git clone - Get the repo initially
  • git status - Check what changed
  • git add - Stage changes
  • git commit -m "message" - Create local commit
  • git push origin main - Push to GitHub
💡 Tip: After pushing, GitHub Pages auto-deploys. Wait 1-2 minutes for changes to appear.

9 Quick Start Checklist

Initial Setup (first time)

  • Install Node.js ≥ 18
  • Install Git
  • Clone repository: git clone ...
  • Install dependencies: npm install
  • Start dev server: npx serve .

For Each Chapter Addition

  • Run npm run check-codes (for language chapters)
  • Update js/subject.js with resourceUrl
  • Update sw.js STATIC_ASSETS
  • Update subject page (add .chapter-link CSS, etc.)
  • Run npm run validate
  • Git commit and push
