📚 Study Dashboard Workflow Guide

Complete guide to adding chapters with resourceUrl, Firestore sync, and cache management

1 Check Available Codes (for Language Chapters)

For English, Malay, or Chinese chapters:

Run this command to see what 3-digit code to use:

npm run check-codes

Output shows:

📊 Chapter Code Status Language | Highest Code | Next Code ---------|--------------|---------- en | en_057 | en_058 my | my_055 | my_056 zh | zh_029 | zh_030
ℹ️ Note: Standard chapters (t2_chapter1.html, t4_force_motion.html) don't need this — use form + chapter number instead.

2 Prepare HTML File

Upload or locate your chapter HTML file:

  • Place in root-level subject folder: chemistry/, english/, etc.
  • Name follows pattern: t4_chapter1.html or en_058.html
  • Ensure it exists on disk before wiring it up

Add Back to Dashboard button (if not present):

<div class="top-nav"> <a class="dashboard-link" href="../index.html"> ← Back to Dashboard </a> </div>

3 Update subject.js

Add resourceUrl to the matching chapter:

{ form:"T4", globalNum: 8, chapterNum: 8, chapter:"Respiratory Systems in Humans and Animals", chinese:"人类与动物的呼吸系统", resourceUrl:"../biology/biology_chapter8.html", // ← ADD THIS status:"Not Started", confidence:0, last_updated:"2026-05-20" }

For language chapters (en_/my_/zh_): Also add chapterCode field:

chapterCode:"en_058"

4 Update Service Worker Cache

Add to STATIC_ASSETS in sw.js:

const STATIC_ASSETS = [ '/', '/index.html', '/css/style.css', '/js/subject.js', '/js/firebase.js', '/subjects/biology.html', '/biology/biology_chapter8.html', // ← ADD THIS // ... other assets ];

Important: Use leading slash, path is from root

5 Update Subject Page (subjects/*.html)

Add CSS styling for chapter links:

.chapter-link{ display:inline-block; margin-top:10px; color:#1d4ed8; font-size:14px; text-decoration:none; } .chapter-link:hover{ text-decoration:underline; }

Add chapterResource variable in render function:

const chapterResource = chapter.resourceUrl ? `<a class="chapter-link" href="${chapter.resourceUrl}">Open notes</a>` : "";

Include in card.innerHTML:

card.innerHTML = ` <div class="chapter-top"> <div class="status-icon ${iconClass}">${icon}</div> <div> <div class="chapter-title"> Chapter ${chapter.globalNum}: ${chapter.chapter} </div> <div class="chapter-cn">${chapter.chinese}</div> ${chapterResource} <!-- ADD THIS --> </div> </div> `;

Add click guard in event listener:

card.addEventListener("click", async (e)=>{ if(e.target.closest(".chapter-link")){ return; } // ← ADD THIS chapter.status = nextStatus(chapter.status); await saveData(); render(); });

Add Firestore sync function:

async function addMissingChineseLabels(){ data.chapters.forEach((ch, idx)=>{ const seed = biologyData.chapters[idx]; if(seed.resourceUrl && !ch.resourceUrl){ ch.resourceUrl = seed.resourceUrl; } if(seed.chinese && !ch.chinese){ ch.chinese = seed.chinese; } }); } // Then in loadData(): if(snap.exists()){ data = snap.data(); await addMissingChineseLabels(); // ← ADD THIS }
⚠️ Critical: Without the Firestore sync function, the link won't appear if the document was saved before resourceUrl was added!

6 Bump Cache Versions

Update version numbers in subject page imports:

<link rel="stylesheet" href="../css/style.css?v=32"> <!-- increment --> import { biologyData } from "../js/subject.js?v=17"; <!-- increment -->

This forces browsers to fetch fresh copies instead of using cached versions.

ℹ️ Tip: Increment both CSS and module import versions each time you make changes.

7 Validate Project

Run validation to catch issues:

npm run validate

This checks:

  • subject.js, subject-registry.js, index.html consistency
  • All chapter pages exist and are properly linked
  • resourceUrl paths are correct
✓ Success: Validation should output "Validated 14 dashboard subjects."

8 Commit & Push

Stage and commit your changes:

git add js/subject.js sw.js subjects/biology.html
git commit -m "feat: add biology chapter 8 with Open notes link"
git push origin main
ℹ️ Commit message format: Describe what you added, not internal details.

📋 Quick Reference Checklist

Before Adding a Chapter

  • Run npm run check-codes (for language chapters)
  • Confirm HTML file exists on disk
  • Add Back to Dashboard button if missing

Code Updates Required

  • Add resourceUrl to chapter in js/subject.js
  • Add path to STATIC_ASSETS in sw.js
  • Add .chapter-link CSS to subjects/subject.html
  • Add chapterResource variable
  • Add click guard in card event listener
  • Add addMissingChineseLabels() function
  • Call sync function in loadData() when snap.exists()
  • Bump cache versions (?v=N)

Final Steps

  • Run npm run validate
  • Commit changes with descriptive message
  • Push to GitHub
  • Wait 1-2 minutes for GitHub Pages to rebuild
  • Hard refresh browser (Ctrl+Shift+R)

🔧 Troubleshooting

Link doesn't appear on GitHub Pages

  • Verify addMissingChineseLabels() function is present
  • Check that it's called in loadData() when snap.exists()
  • Hard refresh browser (Ctrl+Shift+R) to clear cache
  • Wait 2-3 minutes for GitHub Pages to rebuild

Link appears but clicking doesn't navigate

  • Verify resourceUrl path is correct (relative from subjects/)
  • Check that HTML file exists in correct location
  • Ensure path is in STATIC_ASSETS in sw.js

Validation fails

npm run validate

Read error messages carefully — they indicate which files are inconsistent or missing.

💡 Key Concept: Firestore Sync

Why is addMissingChineseLabels() needed?

Subject pages load from Firestore. If a Firestore document was created and saved BEFORE you added the resourceUrl to subject.js, the Firestore doc won't have that field.

The sync function updates Firestore with missing fields from the seed data (subject.js) every time the page loads, ensuring the "Open notes" link always displays.
