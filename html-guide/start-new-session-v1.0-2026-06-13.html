🚀 Start New Claude Code Session

Quick checklist to get oriented and ready to work

1 First Thing: Check Memory

Read your memory files

Your persistent memory holds the project context from all prior sessions. Always check it first:

  • Open: C:\Users\User\.claude\projects\C--development-paul-study-dashboard\memory\MEMORY.md
  • Read the index to understand current project state
  • Key memories: project overview, feedback preferences, technical solutions
💾 Memory files include:
- Project overview & tech stack
- Available commands & workflow
- Firestore sync requirements
- Your preferences (no unprompted push, etc.)

2 Verify Your Environment

Quick system check:

node --version

Should be ≥ 18.0.0

npm --version

Should be ≥ 9.0.0

git --version

Should be latest

ℹ️ If versions are wrong: Install from nodejs.org (includes npm) and git-scm.com, then restart PowerShell.

3 Navigate & Check Status

Set up your working directory:

cd C:\development\paul\study-dashboard
git status

Shows current branch and any uncommitted changes. Should show "On branch main" with no changes (or list what changed since last session).

4 Know Your Commands

Three essential commands:

📋 npm run validate

Cross-checks subject.js, registry, index.html, and all subject pages for consistency. Always run before committing.

npm run validate

🔢 npm run check-codes

Shows next available 3-digit codes for English (en_), Malay (my_), and Chinese (zh_) chapters. Run when adding language chapters.

npm run check-codes

🌐 npx serve .

Local dev server on port 3000. Use to test changes before pushing.

npx serve .

Then open: http://localhost:3000

6 Key Things to Remember

Critical workflow facts:

  • Firestore sync is required: addMissingChineseLabels() function needed when adding chapters with resourceUrl
  • Cache versions bump: Increment ?v=N on CSS and module imports to force fresh loads
  • Git workflow is mandatory: git add → git commit → git push (only push when explicitly asked)
  • Validate before commit: Always run npm run validate to catch inconsistencies
  • Chapter files go in root folders: chemistry/, english/, history/ — NOT under subjects/
