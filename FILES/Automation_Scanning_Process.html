🔎 How It Scans: The Technical Process

Scanning Method: Git Status Command

The agent uses the git status command to find untracked files. These are HTML files you uploaded but haven't been committed yet.

Exact Git Command Used

git status

This command shows output like:

On branch main Untracked files: (use "git add ..." to include what will be committed) subjects/physics/t4_chapter1.html subjects/english/t4_t5_chapter1.html subjects/history/t2_chapter10.html nothing added to commit but untracked files present

Pattern Matching

The agent looks for files matching this regex pattern:

t[0-9](_[0-9])?_chapter[0-9]+\.html

Matches:

  • ✓ t2_chapter1.html (form T2, chapter 1)
  • ✓ t4_chapter10.html (form T4, chapter 10)
  • ✓ t4_t5_chapter1.html (combined form T4/T5, chapter 1)

Does NOT match:

  • ✗ force_motion.html (no form/chapter pattern)
  • ✗ chapter10.html (missing form prefix)
  • ✗ t2_notes.html (missing chapter pattern)

⚙️ Scanning Flow: Step-by-Step

1

Check-in to Repository

Remote agent clones fresh copy of study-dashboard repo from GitHub

2

Run Git Status

Executes git status to find untracked files

$ git status
3

Filter by Pattern

Separates files into:

  • MATCHED: Files following t<form>_chapter<num>.html
  • UNMATCHED: Files not following pattern
4

Process Matched Files

For each matched file:

  • Parse filename → extract form and chapter number
  • Find matching chapter in js/subject.js
  • Add resourceUrl property
  • Update sw.js STATIC_ASSETS
5

Validate & Commit

Run npm run validate then git commit & push

6

Log Execution

Write results to FILES/auto_integration_log.txt with timestamp

7

Commit Log to GitHub

Push log entry so you can verify what happened
