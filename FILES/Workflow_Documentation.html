📚 Study-Dashboard Chapter HTML Automation

Complete Workflow Documentation

📋 Overview

This system automates the integration of new chapter HTML files into the study-dashboard. When you upload a chapter file to a subject directory, the system automatically:

  • Adds the "Back to Dashboard" button (if missing)
  • Updates the subject data with the chapter link
  • Registers the file in the service worker cache
  • Validates the entire project
  • Commits and pushes changes to GitHub
  • Logs all execution details with timestamps

🤖Phase 3: Agent Processing (Every 4 Hours)

Status: AUTOMATED

Execution Schedule

Cron Expression: 0 */4 * * *

00:00 | 04:00 | 08:00 | 12:00 | 16:00 | 20:00

Asia/Kuala_Lumpur Times (Local):

8:00 AM | 12:00 PM | 4:00 PM | 8:00 PM | 12:00 AM | 4:00 AM

Agent Processing Steps

For each uploaded HTML file matching the pattern, the agent:

  1. Check for "Back to Dashboard" button
    • Searches for existing button (class: dashboard-link)
    • If missing, inserts HTML button after <body> tag
    • If missing, adds CSS styling to <head> section
    • Avoids duplicates if button already exists
  2. Parse filename
    • Extract form: t4 → T4, t4_t5 → T4/T5
    • Extract chapter number: chapter1 → 1
  3. Update js/subject.js
    • Find matching chapter by form + chapterNum
    • Add resourceUrl property
    • Example: resourceUrl: "../subjects/physics/t4_chapter1.html"
  4. Update sw.js (Service Worker)
    • Add file path to STATIC_ASSETS array
    • Format: 'subjects/physics/t4_chapter1.html'
    • Prevents duplicate entries
  5. Validate project
    • Run: npm run validate
    • Ensures no errors introduced
  6. Commit & Push to GitHub
    • Stages modified files
    • Creates commit with message: Auto-add chapter notes: t4_chapter1.html
    • Pushes to main branch
    • Only if validation passes
⚠️ VALIDATION FAILURE: If npm validate fails, the agent will NOT commit. You'll see the error in the log file. Fix the issue and reupload the file.

🔧Troubleshooting

File Not Detected

Check:

  • Filename matches pattern: t
    _chapter.html
  • File is in correct directory: subjects//
  • File is untracked (git not aware yet)
  • Wait until next 4-hour execution cycle

Validation Error

Solution:

  • Check log file for error details
  • File won't be committed until error is fixed
  • Fix the issue and reupload the file

Chapter Not Showing in Dashboard

Check:

  • Verify commit was pushed to GitHub
  • Hard refresh browser (Ctrl+Shift+R)
  • Clear service worker cache if needed
  • Check browser console for errors

Filename Suggestion Not Matching

What to do:

  • Review the suggestion in the log file
  • Check your HTML file content for clues
  • Rename and reupload with suggested name
