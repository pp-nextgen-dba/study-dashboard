Study-Dashboard — Chapter Auto-Integration

How to upload chapter notes and let the workflow wire everything up automatically

Overview

When you push a new chapter HTML file to GitHub, this workflow automatically:

  • Adds the Back to Dashboard button to the chapter page (if missing)
  • Adds resourceUrl to the matching chapter in js/subject.js
  • Adds the file to STATIC_ASSETS in sw.js (service worker cache)
  • Ensures the subject page has the Open notes link and Firestore sync logic
  • Validates, commits and pushes all changes to GitHub
  • Appends a timestamped entry to FILES/auto_integration_log.txt

Step 1 — Upload Your HTML File to GitHub

Where to put the file

Drop it directly into the root-level subject folder — not inside subjects/:

chemistry/t4_chapter2.html physics/t4_chapter1.html history/t2_chapter5.html rbt/t2_chapter3.html english/t4_t5_chapter2.html

No subfolders. No archive/. Flat files only.

Filename pattern

PatternExampleMeans
t<form>_chapter<num>.htmlt4_chapter2.htmlForm 4, Chapter 2
t<form>_t<form>_chapter<num>.htmlt4_t5_chapter1.htmlForm 4/5 combined, Chapter 1

The chapter number in the filename is the position within that form (1st T4 chapter = 1, 2nd = 2, etc.).

How to upload — two options

Option A — GitHub web UI (easiest):

  1. Go to your repo on GitHub
  2. Navigate to the subject folder (e.g. chemistry/)
  3. Click Add file → Upload files
  4. Drop the HTML file in → commit directly to main

Option B — Push from local machine:

git add chemistry/t4_chapter2.html git commit -m "Add chemistry T4 chapter 2 notes" git push origin main

Note on this Claude Code environment

This workflow runs in a remote container that clones from GitHub. Files only on your local machine are invisible here. Always push to GitHub first — the workflow fetches from there.

Step 2 — Trigger the Workflow

How to run

Open this Claude Code session and say:

run workflow once now

That's it. The workflow runs immediately and reports what it did.

What happens when triggered

1. Pull — fetches latest from GitHub (git pull origin main)
↓
2. Scan — finds all HTML files in root-level subject folders (chemistry/, physics/, history/, rbt/, english/, etc.)
↓
3. Detect — cross-checks each file against js/subject.js; any chapter missing resourceUrl gets processed
↓
4. Process — for each new file: adds Back to Dashboard button, adds resourceUrl to subject.js, adds path to sw.js STATIC_ASSETS
↓
5. Subject page check — verifies the subject's subjects/<name>.html has all four required components for "Open notes" to work (see below)
↓
6. Validate — runs npm run validate; aborts commit if errors found
↓
7. Commit & push — commits all changes to GitHub, appends run result to log

What the Workflow Checks on Subject Pages

For every subject that gets a new resourceUrl, the workflow verifies subjects/<name>.html has all four of these. Missing ones are added automatically.

1 — .chapter-link CSS

Styles the "Open notes" anchor link in chapter cards.

.chapter-link{display:inline-block;margin-top:10px;color:#1d4ed8;font-size:14px;text-decoration:none} .chapter-link:hover{text-decoration:underline}

2 — chapterResource variable in card render

Generates the "Open notes" link HTML before it's inserted into the card.

const chapterResource = chapter.resourceUrl ? `<a class="chapter-link" href="${chapter.resourceUrl}">Open notes</a>` : ""; // then inside card.innerHTML: ${chapterResource}

3 — Click guard in card click handler

Prevents clicking "Open notes" from also toggling the chapter's status.

card.addEventListener("click", async (e) => { if(e.target.closest(".chapter-link")){ return; } // ... rest of status toggle });

4 — addMissingChineseLabels() Firestore sync

Subject pages read chapter data from Firestore, not directly from subject.js. If the Firestore document was saved before resourceUrl was added, it won't have the field — meaning the "Open notes" link won't appear even though the code is correct.

This function runs on page load and syncs any missing resourceUrl (and chinese) fields from the seed data back into Firestore automatically.

// Called inside loadData() when snap.exists(): if(addMissingChineseLabels(data)){ data.last_saved = new Date().toISOString(); await setDoc(docRef, data); }

Troubleshooting

"Open notes" link not visible after workflow ran

  • Hard-refresh the subject page: Ctrl+Shift+R (clears service worker cache)
  • If still missing: the subject page may be missing the addMissingChineseLabels function — tell Claude to check and fix it
  • Root cause: Firestore had a stale document saved before resourceUrl was added

File not detected by workflow

  • Confirm file is pushed to GitHub (not just saved locally)
  • Confirm filename matches t<form>_chapter<num>.html
  • Confirm file is in the root-level subject folder (e.g. chemistry/), not inside subjects/ or a subfolder
  • Check that resourceUrl is not already set in js/subject.js for that chapter (workflow skips already-integrated files)

Validation error — workflow did not commit

  • Check the output reported in chat for the specific error
  • Fix the issue (usually a mismatch in subject.js), then re-run the workflow
